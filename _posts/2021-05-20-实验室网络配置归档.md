## WSL Ubuntu 20.04 LTS

WSL确实是个好文明，至少方便的一批。解决了许多需要切换系统才能做的事情。安装WSL有很多方式[^1]，可以参与windows体验版一键安装，也可以进入微软商店直接购买Ubuntu 20.04 LTS

这里我安装选择了最麻烦的按部就班方式，具体为

### 1. 启动子系统可选功能

以管理员身份打开 PowerShell 并运行：

```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```

### 2. 启用虚拟化功能

以管理员身份打开 PowerShell 并运行：

```powershell
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

### 3. 下载 Linux 内核更新包

[点此下载](https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi)

### 4. 将 WSL 2 设置为默认版本

以管理员身份打开 PowerShell 并运行：

```powershell
wsl --set-default-version 2
```

### 5. 安装指定版本的Linux分发版

打开微软商店选择适合的版本即可

安装后运行指定版本的Linux，我们就可以直接在windows上运行Linux系统，效果如图（以Ubuntu 20.04 LTS为例）

## 配置Windows OpenSSH Server

对于Windows 10的当前版本而言，可以直接在 **设置/应用/可选功能** 中选择安装OpenSSH的客户端和服务端。为了方便起见，两个均进行安装。由于学校电脑均为内网机，因此可以直接采用ssh命令对内网ip进行访问，例如

```bash
ssh usrname@ipaddress
```

我们可以用这个命令进行简单的测试，连接成功即可

由于我的数据集文件采用CPU服务器创建，训练过程由GPU服务器进行，中间需要一定的文件交换，此时可以使用`scp`命令

```bash
scp usrname@ipaddress:<dir1> <dir2>
```

这是其中一种操作示例，从网络机中的一个设备拷贝到本地设备，除此之外还有比较多的用法，可以类推所以不做更多介绍

## VS Code SSH远程设置

使用ssh可以直接访问一台网络上的计算机，那么很容易通过一些熟练的操作对远程计算机进行控制，但是没有图形界面的效率还是略低，因此采用VS Code进行远程操作

安装`Remote - SSH`扩展，左边栏会增加一个SSH远程的框（实际上还可以选择`Remote - WSL`，这样左边框可以选）。按照VS Code提示的步骤反复进行几次操作即可实现远程。我们选择指定文件夹之后，再用VS Code时就相当于在本机上操作，不太方便的地方就是不能像RDP协议一样显示图片之类的，但是借助VS Code的功能我们可以查看输出的图像文件。
